# 趋势突破前高有效性选股策略（主板）

> 本策略是一个**二次筛选策略**，不负责发现突破动作。
> 已有初始公式负责把近期发生过突破的股票放入观察池；本文策略只回答一个问题：
> **观察池里的这次突破，是否已经被市场确认有效，并且当前仍适合作为选股结果？**

本文档定义策略的理论边界、突破事件锚点、状态机、有效突破类型、量价关系、失效条件及最终输出语义，
作为后续编写通达信/同花顺选股公式或 Python 事件跟踪实现的依据。

---

## 一、策略定位：突破发现与有效性判断分离

完整工作流分为两个阶段：

```text
第 0 层：初始突破公式（已有）
发现突破动作 → 建立突破事件 → 进入观察池
                         │
                         ▼
第 1 层：本文有效性策略
固定跟踪本次突破 → 判断 Pending / Valid / Failed / Expired
                         │
                         ▼
最终选股结果：ValidActive
```

两个阶段采用不同目标：

| 阶段 | 目标 | 设计倾向 |
|---|---|---|
| 初始突破公式 | 尽量完整地发现突破动作 | 高召回，避免提前用量能漏掉潜在突破 |
| 有效性策略 | 判断市场是否真正接受突破后的价格 | 高精度，结合点位、时间与量价行为 |

### 1.1 初始突破公式负责什么

初始公式负责确认以下背景与动作：

- 股票处于预设的上升趋势背景；
- 存在符合要求的阶段前高；
- 当日收盘明确越过该前高；
- 这是本轮首次发生的突破动作；
- 股票进入观察池并保留若干交易日。

初始公式只认定“**突破动作已经发生**”，不认定“突破已经有效”。

### 1.2 有效性策略负责什么

本文策略负责跟踪突破后的市场行为：

- 价格是否接受前高上方的新交易区间；
- 原阻力是否开始转化为支撑；
- 成交量反映的是需求扩张、供给收缩，还是卖压增强；
- 本次突破是有效、失败，还是一直没有得到确认；
- 即使突破有效，当前是否已经过度延伸。

### 1.3 核心原则

> **突破是一个事件，有效是后续市场行为形成的结论。**

因此，初始突破不应强制要求放量。普通量、缩量、涨停或跳空突破都可以先进入观察池，
再由本文策略根据后续证据分类确认。

---

## 二、核心理论：阻力突破后的价格接受

前高附近通常聚集历史套牢盘、获利盘和心理锚定，是潜在卖压区域。
价格收盘越过前高，只能说明买方在突破日暂时占优；只有后续市场继续在前高上方成交，
原阻力才可能转化为支撑。

策略的理论链条是：

```text
阶段阻力存在
    ↓
收盘突破阻力
    ↓
市场在阻力上方持续成交
    ↓
需求扩张或供给收缩得到验证
    ↓
阻力转化为支撑
    ↓
突破确认有效
```

有效性判断包含两个不可替代的维度：

1. **价格接受**：价格能否站在前高上方，或者回踩后重新收回前高。
2. **供需证据**：推动或维持突破的是新增买盘、卖盘减少，还是仅仅一次短暂冲高。

价格接受是必要条件。成交量只能解释价格行为，不能替代价格本身。

## 三、关键概念定义

### 3.1 什么是前高

前高不是单纯的数学最高价，而是市场曾经明显遇到抛压的**阶段阻力区域**。

按理论质量从高到低分为：

| 类型 | 定义 | 理论质量 |
|---|---|---|
| 平台前高 | 价格多次接近同一区域但未能有效突破 | 高 |
| 波段前高 | 形成明显局部高点，之后发生充分回落或整理 | 中 |
| 孤立尖峰 | 单日冲高、异常长上影形成的最高价 | 低 |

当前初始公式以一定回看窗口内的最高价近似前高，这是可执行的第一版定义；
后续可以增加平台持续时间、触碰次数、振幅收敛和异常上影过滤，提高前高质量。

### 3.2 前高是一段区域，不是一条绝对精确的线

定义：

- `G0`：突破日所突破的前高上边界；
- `G_SUPPORT`：前高支撑区下边界；
- `[G_SUPPORT, G0]`：突破后的允许回踩区域。

价格短暂进入该区域，不等于突破失败；明确跌破支撑区下边界，才属于强价格失效。

### 3.3 什么是突破动作

突破动作是价格从阻力区域下方进入上方，可以分为：

| 行为 | 定义 | 是否进入本文跟踪 |
|---|---|---|
| 盘中突破 | 最高价越过前高，但收盘重新回到阻力区内 | 否，仅算突破尝试 |
| 收盘突破 | 收盘价明确站到前高上方 | 是，建立突破事件 |
| 有效突破 | 后续价格和供需行为证明市场接受了新价格区间 | 本文的判定结果 |

### 3.4 什么是价格接受

价格接受是指市场愿意持续在前高上方成交，而不是短暂冲过后立即回落。

理论上至少应满足：

- 当前收盘位于 `G0` 上方；
- 突破后的多数关键收盘位于 `G0` 上方；
- 没有明确跌破 `G_SUPPORT`；
- 没有跌破突破日关键低点；
- 突破后不是单日冲高、随后立即回到原平台。

公式实现时，可以用“最近若干日站上前高的天数”“突破日至今最低收盘”和“当前收盘位置”表达。

## 四、突破事件与固定锚点

有效性判断必须围绕一次确定的突破事件展开。股票进入观察池时，应固定以下信息：

| 锚点 | 含义 | 用途 |
|---|---|---|
| `D0` | 突破日 | 计算事件年龄和观察窗口 |
| `G0` | D0 所突破的前高 | 判断站稳、回踩和破位 |
| `G_SUPPORT` | 前高支撑区下边界 | 区分浅回踩与明确失守 |
| `O0/H0/L0/C0` | 突破日 OHLC | 判断突破日质量及结构根基 |
| `V0` | 突破日成交量 | 判断突破日量能类型 |
| `V_BASE` | 突破日前的正常成交量水平 | 作为放量/缩量比较基准 |
| `VOLATILITY0` | 突破时的波动率 | 约束支撑容忍度和过度延伸 |

### 4.1 锚点不可漂移

进入观察池后，必须始终使用 D0 当天确定的 `G0` 和 `L0`。
不能每天重新计算一个新的前高，再用新前高判断旧突破是否有效。

### 4.2 重复突破的处理

同一只股票在观察期内可能再次突破更高位置：

- 如果旧事件仍在跟踪，应保留旧事件的结论；
- 新突破应建立新的事件和新的 `D0/G0`；
- 公式受限时可以只近似跟踪最近一次突破，但理论定义必须按事件区分；
- 同一个事件一旦失败，不能因为价格次日拉回就恢复。后续重新突破属于新事件。

## 五、状态机

建议采用以下状态：

```text
                         ┌──────────────┐
突破事件发生 ──────────→ │ Observed     │  突破刚发生
                         └──────┬───────┘
                                ▼
                         ┌──────────────┐
                         │ Pending      │  等待确认
                         └───┬─────┬────┘
                    确认有效 │     │ 明确失败
                             ▼     ▼
                 ┌──────────────┐  ┌──────────────┐
                 │ Valid        │  │ Failed       │
                 └──────┬───────┘  └──────────────┘
                        │
             ┌──────────┼──────────┐
             ▼          ▼          ▼
        ValidActive  Extended  Invalidated

Pending 超过最大观察期仍未确认 → Expired
```

状态定义：

| 状态 | 定义 | 是否作为最终选股结果 |
|---|---|---|
| `Observed` | D0 刚发生突破，尚无后续数据 | 否 |
| `Pending` | 未失败，但价格接受或供需证据不足 | 否，继续观察 |
| `Valid` | 本次突破已得到有效确认 | 中间状态 |
| `ValidActive` | 突破有效，且当前尚未过度延伸 | 是 |
| `Extended` | 突破有效，但价格已经距离突破位过远 | 否，保留标记 |
| `Failed` | 本次突破被市场明确否定 | 否，当前事件永久失效 |
| `Expired` | 未明显失败，但在观察期内始终没有确认 | 否，超时移出 |
| `Invalidated` | 曾经有效，后来结构被破坏 | 否 |

这个状态机区分了三个经常被混淆的问题：

- `Failed`：有明确反证；
- `Expired`：没有明确反证，但突破缺乏效率；
- `Extended`：突破是有效的，只是当前不再适合作为新候选。

## 六、有效突破的共同条件

任何一种有效路径都必须满足以下共同条件：

1. 存在固定的突破事件和 `D0/G0`；
2. 已经过最小确认时间，不能只凭 D0 当天直接下结论；
3. 当前仍在最大观察窗口内；
4. 价格接受成立；
5. 至少出现一种可解释的正向供需证据；
6. 未发生任何强失效条件。

可概括为：

```text
突破有效 = 价格接受
         AND 正向供需证据
         AND 时间确认
         AND NOT 失效
```

其中“正向供需证据”不等于“必须放量”，而是由下面四条路径之一提供。

## 七、有效突破的四条确认路径

### 7.1 类型 A：放量突破后站稳

特征：

- D0 相对突破前正常成交量明显放大；
- D0 收盘位于 `G0` 上方；
- D0 收盘位置较强，没有明显冲高回落；
- 后续数日没有跌破前高支撑区；
- 经过确认时间后，当前价格仍被市场接受。

理论解释：新增买盘扩张，阻力位附近的卖盘被主动吸收。

注意：D0 放量只是初始证据。如果随后迅速跌回前高下方，仍然属于失败突破。

### 7.2 类型 B：后置放量确认

特征：

- D0 成交量普通或偏低，但已经收盘突破；
- 突破后价格没有明显跌回原平台；
- 后续某日出现正向放量推进；
- 放量时价格位于 `G0` 上方，且收盘质量较好；
- 观察期内没有发生结构失效。

理论解释：突破当天参与度不足，但后续买方资金逐步跟进，补足了需求扩张证据。

这是初始突破公式不应增加量能闸门的主要原因。

### 7.3 类型 C：缩量回踩确认

特征：

- D0 突破已经发生；
- 后续最低价或收盘价回到前高支撑区域；
- 回踩时成交量相对突破前基准或突破日明显下降；
- 收盘没有明确跌破 `G_SUPPORT`；
- 当前重新收回或继续站在 `G0` 上方。

理论解释：突破后获利盘得到消化，但卖压没有扩大；原阻力附近出现承接，开始转化为支撑。

“缩量”必须与“回踩关键位置且守住/收复”同时存在。单独缩量下跌不是正面信号。

### 7.4 类型 D：供给收缩确认

特征：

- D0 和后续没有出现典型放量；
- 价格仍连续维持在 `G0` 上方；
- 回落幅度有限，回落时成交量逐步收缩；
- 没有放量冲高回落或放量破位；
- 经过确认窗口后仍保持价格接受。

理论解释：推动突破的力量不是买盘急剧扩张，而是愿意卖出的筹码减少。

缩量涨停、跳空突破属于供给收缩的特殊形态，但应额外标记：

- 收盘位于法定涨停价或形成明显跳空，只能说明结构强；
- 日线收盘等于最高价，不能单独证明全天“封死”；
- 连续一字涨停可以判定结构强，但应与普通可交易候选区分。

## 八、量价配合定义

### 8.1 成交量基准

放量和缩量应相对于**突破发生前**的正常成交量水平判断，避免把当日成交量本身纳入基准后稀释比较结果。

成交量基准可采用：

- 突破日前若干交易日平均成交量；
- 较长窗口的平均量或中位数；
- 后续公式根据数据稳定性选择具体实现。

理论上，基准必须只使用当时已经存在的数据，不能引用未来成交量。

### 8.2 正向放量

放量只有与正向价格行为同时出现，才能作为有效突破证据：

- 收盘位于 `G0` 上方；
- 收盘不低于开盘，或至少表现出明显价格推进；
- 收盘位于当日波动区间的较高位置；
- 没有明显长上影或冲高回落；
- 成交量相对突破前基准增加。

放量本身只代表分歧或交易活跃，不能自动解释为买方资金流入。

### 8.3 健康缩量

缩量只有在以下背景中才构成正向证据：

- 发生在突破后的回踩或窄幅整理中；
- 回落幅度有限；
- 没有明确跌破前高支撑区；
- 当前已经止跌、收复或继续站稳 `G0`。

它反映的是卖压收缩，而不是简单的“成交量越小越好”。

### 8.4 负面放量

以下量价组合应视为负面证据：

- 放量冲高回落；
- 放量长上影；
- 放量跌回 `G0` 下方；
- 成交量显著放大，但价格无法继续推进；
- 突破后高位放量滞涨。

这些行为只能说明市场分歧扩大或卖压增强，不应笼统归因于“主力出货”。

### 8.5 量价判定矩阵

| 价格行为 | 成交量 | 理论判断 | 状态倾向 |
|---|---|---|---|
| 强势突破并收在高位 | 放量 | 需求扩张 | Pending → Valid 候选 |
| 强势突破并收在高位 | 平量/缩量 | 可能供给收缩，也可能参与不足 | Pending |
| 突破后继续站稳并推进 | 正向放量 | 后置需求确认 | Valid 候选 |
| 回踩前高支撑区并守住 | 缩量 | 卖压收缩、回踩健康 | Valid 候选 |
| 回踩前高支撑区并守住 | 放量 | 分歧较大，尚不能直接判好 | Pending |
| 放量但冲高回落 | 放量 | 价格没有接受高位 | Pending / Failed |
| 跌回 `G0` 下方 | 放量 | 强负面，卖压增强 | Failed |
| 浅度跌回 `G0` 附近 | 缩量 | 可能是正常回踩 | Pending |
| 明确跌破支撑区 | 任意 | 价格接受失败 | Failed |

## 九、突破失效与超时定义

### 9.1 当日拒绝

盘中越过前高，但收盘重新回到阻力区内，尤其伴随长上影或明显放量。

严格来说，这种情况不应建立正式突破事件；它属于“突破尝试失败”。

### 9.2 强价格失效

发生以下任一情况，可以判定本次事件失败：

- 收盘明确跌破 `G_SUPPORT`；
- 收盘跌破突破日最低价 `L0`；
- 连续多个交易日收在 `G0` 下方，表明市场持续拒绝突破后的价格。

### 9.3 放量破位

价格收回 `G0` 下方，同时成交量明显增加。

这是比普通缩量回踩更强的负面证据。即使跌破幅度不大，也说明前高没有稳定转化为支撑。

一旦放量破位发生，本次突破事件永久进入 `Failed`，不能因为次日拉回而恢复。

### 9.4 时间失效

在最大观察期内：

- 没有形成价格接受；
- 没有正向放量推进；
- 没有形成健康回踩或供给收缩确认；
- 同时也没有明确破位。

这种情况应定义为 `Expired`，而不是 `Failed`。它表示突破缺乏效率和确认，不代表一定是假突破。

### 9.5 有效后的结构破坏

已经进入 `Valid` 的事件，后续仍可能跌破 `G_SUPPORT` 或 `L0`。
此时状态应从 `Valid` 转为 `Invalidated`，而不是修改历史结论为“从未有效”。

## 十、有效突破与最终入选分离

一只股票的突破可能已经有效，但当前不再适合作为新的选股结果，例如：

- 已经连续快速上涨；
- 当前价格距离 `G0` 过远；
- 当前价格距离趋势均线过远；
- 出现连续无法正常成交的涨停；
- 上方很近仍有更高级别阻力。

因此必须区分：

```text
突破有效 = 本次突破结构成立

最终入选 = 突破有效
         AND 当前仍在合理延伸范围内
         AND 不属于异常交易状态
```

建议以价格相对 `G0` 的距离和相对波动率共同判断是否过度延伸。
固定百分比便于公式实现，波动率归一化更适合不同波动特征的股票。

## 十一、公式化判定框架

本文暂不锁定具体公式语法，但所有理论定义都应能映射到日线 OHLCV 条件。

### 11.1 基础事件条件

```text
在池中       := 初始突破事件仍处于观察窗口
事件年龄     := 当前交易日距离 D0 的交易日数量
价格接受     := 当前站在 G0 上方，且突破后未发生强失效
```

### 11.2 四条有效路径

```text
路径A := D0正向放量 AND 后续站稳
路径B := D0未放量 AND 窗内出现正向放量推进 AND 后续站稳
路径C := 窗内出现缩量回踩支撑区 AND 当前收复/站稳G0
路径D := 窗内始终维持价格接受 AND 回落缩量 AND 无负面放量
```

### 11.3 失效条件

```text
价格失效 := 明确跌破支撑区 OR 跌破L0 OR 持续收在G0下方
量价失效 := 历史上曾发生放量破位
失效     := 价格失效 OR 量价失效
```

### 11.4 状态判定

```text
Valid   := 在池中
           AND 已过最小确认时间
           AND 价格接受
           AND (路径A OR 路径B OR 路径C OR 路径D)
           AND NOT 失效

Pending := 在池中 AND NOT Valid AND NOT 失效
Failed  := 在池中 AND 失效
Expired := 超过最大观察期 AND 历史上从未Valid AND NOT Failed
```

### 11.5 最终选股输出

```text
ValidActive := Valid
               AND 当前未过度延伸
               AND 当前收盘仍在G0上方
               AND 不属于异常交易状态
```

正式选股公式应只输出 `ValidActive`。`Pending/Failed/Expired/Extended` 用于观察池管理、展示或排除。

## 十二、参数角色与第一版建议

参数值不是理论真理，只是将理论映射为公式时的第一版起点，必须通过历史样本和不同市场环境验证。

| 参数角色 | 当前建议 | 说明 |
|---|---:|---|
| 前高回看窗口 | 60 个交易日 | 近似季度级阶段阻力 |
| 前高最小年龄 | 5 个交易日 | 排除刚形成的短期新高 |
| 初始突破缓冲 | 0.5% | 过滤擦边触碰，后续可加入波动率归一化 |
| 最小确认时间 | 3 个交易日 | 避免把 D0 一日表现直接认定为有效 |
| 当前观察池窗口 | D0～D5 | 与已有 `K=5` 初始公式一致，属于快速确认版本 |
| 支撑容忍区 | 前高下方约 3% | 区分浅回踩与明确失守，后续可改为波动率口径 |
| 放量阈值 | 突破前基准量的约 1.3 倍 | 必须配合正向价格行为 |
| 缩量阈值 | 突破前基准量的约 0.8 倍 | 仅用于回踩或整理场景 |

当前观察池只保留 D0～D5，因此最小确认时间为 3 日时，实际确认机会集中在 D3～D5。
如果希望覆盖更慢的后置放量和回踩确认，应延长观察池保留时间，而不是在第二阶段暗中跟踪已经出池的事件。

## 十三、边界情况

### 13.1 缩量普通突破

先进入 `Pending`。它可以通过后置放量、缩量回踩或持续供给收缩获得确认，
不能因为 D0 未放量直接判失败，也不能只因没有跌破直接判有效。

### 13.2 缩量涨停或跳空突破

作为供给收缩的特殊事件处理。需要后续价格接受确认，并单独标记连续一字或成交受限风险。

### 13.3 放量突破后立即跌回

D0 放量不能覆盖后续失败。如果跌回支撑区下方或发生放量破位，应判 `Failed`。

### 13.4 浅度跌回前高下方

如果仍在支撑容忍区内且成交量收缩，保持 `Pending`；只有重新收回 `G0` 并满足有效路径后，才能升级为 `Valid`。

### 13.5 多次突破

理论上按事件分别记录。公式只能锚定最近一次时，应明确这是实现近似，不能据此修改事件理论。

### 13.6 停牌、除权和异常行情

停牌会破坏观察日计数，除权会改变历史价格可比性。公式和 Python 实现都应保证使用一致的交易日序列和复权口径，
并对新股、风险警示、退市整理及异常交易状态单独过滤。

---

## 十四、理论总纲

本策略可以概括为：

> 初始公式在健康趋势背景中发现收盘突破阶段前高的事件，并将其纳入观察池；
> 有效性策略固定锚定突破日 `D0` 和前高 `G0`，观察市场是否持续接受前高上方的新价格，
> 再通过放量突破后站稳、后置放量、缩量回踩或供给收缩四条路径之一确认供需关系；
> 对明确破位、放量破位和超时未确认分别标记为失败或过期；
> 最终只输出突破已经有效、当前仍站在前高上方且尚未过度延伸的 `ValidActive` 标的。

## 十五、接入选股工作台（引擎兼容版）

本项目自带公式选股工作台（`src/core/screener/` + `/api/screener`），可直接承载本策略的**分层选股**。
工作台执行模型：一条公式 → 逐只股票跑 K 线 → **取最后一根布尔值** = 是否入选；
选股范围 = 自选股 + 关注板块成分股（上限 300），默认取 120 天 K 线；**无状态**，每次运行独立重算。

→ 因此本策略落成 **3 个独立保存的公式**：观察池 / 有效 / 无效，分别运行。

### 15.1 为接入所做的引擎改造

原公式语法与引擎不兼容，已对 `src/core/screener/formula.py` 做如下增强（向后兼容）：

| 改动 | 说明 |
|---|---|
| 新增 `HHVBARS(X,N)` | N 日内最高值到当前的周期数（前高年龄 `DIST` 需要） |
| 新增 `CODELIKE('300')` | 按股票代码前缀过滤；求值器接入 symbol，自动去除交易所字母前缀 |
| 新增字符串字面量 `'...'` / `"..."` | 供 `CODELIKE` 使用 |
| 变量周期空值降级 | `BARSLAST` 从未成立（`None`）时，`REF/HHV/LLV/COUNT/EXIST` 等优雅返回空序列而非崩溃 |

> 注意：引擎只评估**最后一根**，变量周期函数取最后一根的周期值作标量——这恰好让
> `REF(G,BL)`、`LLV(C,BL+1)`、`EXIST(BIGVOL,BL+1)` 等「锚定到突破日」的逻辑对**今天这一根**算得正确。
> 但也因此，这些锚定公式**不适合用回测引擎逐根回放**（非末根会用错周期）；要回测有效率须走 Python 事件实现。

### 15.2 三条工作台公式

变量名改 ASCII、注释用 `//`、双创过滤用 `CODELIKE` 内联。已通过单测与构造场景验证
（放量突破站稳 → 有效；放量破位 → 无效；双创被过滤）。

**① 观察池**
```
N:=60; M:=5; K:=5; BR:=1.005; DEV:=1.25;
MA5:=MA(C,5); MA10:=MA(C,10); MA20:=MA(C,20); MA60:=MA(C,60);
UPT:=MA5>=MA10 AND MA10>=MA20 AND MA20>=MA60 AND MA60>=REF(MA60,10) AND C>MA20 AND C/MA20<DEV;
G:=REF(HHV(H,N),1);
GB:=REF(HHVBARS(H,N),1)+1; DIST:=GB>=M;
TP:=C>=G*BR AND REF(C,1)<G*BR AND DIST AND UPT;
NO2C:=NOT(CODELIKE('300')) AND NOT(CODELIKE('301')) AND NOT(CODELIKE('688')) AND NOT(CODELIKE('689'));
XG:COUNT(TP,K+1)>0 AND NO2C;
```

**② 有效（ValidActive 近似，作为选股结果）**
```
N:=60; M:=5; K:=5; BR:=1.005; DEV:=1.25; CW:=3; FB:=0.97; RUP:=1.3;
MA5:=MA(C,5); MA10:=MA(C,10); MA20:=MA(C,20); MA60:=MA(C,60);
UPT:=MA5>=MA10 AND MA10>=MA20 AND MA20>=MA60 AND MA60>=REF(MA60,10) AND C>MA20 AND C/MA20<DEV;
G:=REF(HHV(H,N),1);
GB:=REF(HHVBARS(H,N),1)+1; DIST:=GB>=M;
TP:=C>=G*BR AND REF(C,1)<G*BR AND DIST AND UPT;
INPOOL:=COUNT(TP,K+1)>0;
BL:=BARSLAST(TP);
G0:=REF(G,BL); LOW0:=REF(L,BL); LCLOSE:=LLV(C,BL+1);
BIGVOL:=V>=MA(V,5)*RUP;
ZTSEAL:=C>=REF(C,1)*1.097 AND C=H;
VWIN:=EXIST(BIGVOL,BL+1);          // 窗内放量(后置放量确认)
ZTBRK:=REF(ZTSEAL,BL);             // 突破日涨停封死(缩量豁免)
VOLOK:=VWIN OR ZTBRK;
FAIL:=LCLOSE<G0*FB OR LCLOSE<LOW0 OR (C<G0 AND BIGVOL);
NO2C:=NOT(CODELIKE('300')) AND NOT(CODELIKE('301')) AND NOT(CODELIKE('688')) AND NOT(CODELIKE('689'));
XG:INPOOL AND VOLOK AND BL>=CW AND NOT FAIL AND C>=G0 AND NO2C;
```

**③ 无效（用于剔除/排除）**
```
N:=60; M:=5; K:=5; BR:=1.005; DEV:=1.25; FB:=0.97; RUP:=1.3;
MA5:=MA(C,5); MA10:=MA(C,10); MA20:=MA(C,20); MA60:=MA(C,60);
UPT:=MA5>=MA10 AND MA10>=MA20 AND MA20>=MA60 AND MA60>=REF(MA60,10) AND C>MA20 AND C/MA20<DEV;
G:=REF(HHV(H,N),1);
GB:=REF(HHVBARS(H,N),1)+1; DIST:=GB>=M;
TP:=C>=G*BR AND REF(C,1)<G*BR AND DIST AND UPT;
INPOOL:=COUNT(TP,K+1)>0;
BL:=BARSLAST(TP);
G0:=REF(G,BL); LOW0:=REF(L,BL); LCLOSE:=LLV(C,BL+1);
BIGVOL:=V>=MA(V,5)*RUP;
FAIL:=LCLOSE<G0*FB OR LCLOSE<LOW0 OR (C<G0 AND BIGVOL);
XG:INPOOL AND FAIL;
```

> ② 比第七章基础版多了 `C>=G0`（当前仍站在前高上方，近似 `ValidActive` 的「未跌回」），
> 「过度延伸」仍由已有的 `C/MA20<DEV` 兜底。

### 15.3 使用步骤

1. 设置 universe：自选股 / 关注板块，`max_symbols ≤ 300`，`days ≥ 120`（保证 MA60 与 `REF(MA60,10)` 有数据）。
2. 在工作台新建 3 条公式（`StockScreenerFormula`），分别粘贴 ①②③。
3. 日常：跑 ① 维护观察池；跑 ② 得当日候选；跑 ③ 做排除核对。

### 15.4 工作台能力边界（与完整规格的差距）

| 能力 | 工作台 | 完整规格（需 Python 事件层） |
|---|---|---|
| 今日是否 Valid/Failed 形态 | ✅ | ✅ |
| 持久化事件状态、永久 Failed 不恢复 | ❌（无状态重算） | ✅ `StrategySignalRun` |
| `Invalidated` / 真正 `Expired` / 有效率统计 | ❌ | ✅ |
| 历史放量破位、逐事件独立跟踪 | ❌（只抓当天） | ✅ |
| 全市场 | ❌（≤300，自选+关注板块） | ✅ |

→ 工作台用于**今日快照式三层筛选**；完整状态机（§五~九）与有效率统计须落到 `src/core/signals/` + `StrategySignalRun`。

## 附录 A：已有初始观察池公式

该公式是第 0 层，只负责发现突破动作并维护观察池，不负责判断突破是否有效。

```text
N:=60;
M:=5;
K:=5;
BR:=1.005;
DEV:=1.25;

MA5:=MA(C,5);
MA10:=MA(C,10);
MA20:=MA(C,20);
MA60:=MA(C,60);

UPTREND:=MA5>=MA10
         AND MA10>=MA20
         AND MA20>=MA60
         AND MA60>=REF(MA60,10)
         AND C>MA20
         AND C/MA20<DEV;

{近N天前高，不包含今天}
G:=REF(HHV(H,N),1);

{排除创业板、科创板}
去创业板:=NOT(CODELIKE('300')) AND NOT(CODELIKE('301'));
去科创板:=NOT(CODELIKE('688')) AND NOT(CODELIKE('689'));
去双创:=去创业板 AND 去科创板;

{前高距离今天至少M天}
GB:=REF(HHVBARS(H,N),1)+1;
DIST:=GB>=M;

{突破动作：收盘突破近N天前高；不增加量能闸门}
TP:=C>=G*BR
    AND REF(C,1)<G*BR
    AND DIST
    AND UPTREND;

{观察池：今天或过去K个交易日内发生过突破动作}
XG:COUNT(TP,K+1)>0 AND 去双创;
```

## 附录 B：旧版有效性公式草案状态

此前的有效性公式草案只实现了“窗内放量 + 点位未失守”的简化模型，尚未完整表达本文理论，
存在以下缺口，因此不再作为当前实现基准：

1. 未区分正向放量与放量冲高回落；
2. 未真正实现缩量回踩确认；
3. 未覆盖无放量但供给持续收缩的有效路径；
4. 放量破位没有按事件永久保存，可能出现失败后重新恢复；
5. 只锚定最近一次突破，无法严格区分多个事件；
6. 未区分 `Failed`、`Expired`、`Extended` 和 `Invalidated`；
7. 未将“突破有效”与“当前仍适合入选”分开。

后续编写第 1 层有效性公式时，应以本文第十一节的判定框架为准，逐项明确公式语言能够精确实现的条件和只能近似实现的条件。
